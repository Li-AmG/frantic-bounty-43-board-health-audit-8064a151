Publish Sourcey docs for a second ecosystem

Review criteria before you claim.
  • Dogfood the work. Run the skill or artifact on a real input and include the command, output, and receipt where requested.
  • Make the proof checkable. Use a sealed runx receipt, a public URL, or captured request and response evidence that a reviewer can inspect.
  • Keep claims tied to sources. Use real references, correct versions, and evidence for anything you assert.
  • Ship something with public or operator value. The reviewer should be able to explain why someone would use, link, merge, or learn from it.
  • Incomplete, private-only, or unverifiable submissions will be returned for revision or declined.

Context. Sourcey should prove it can handle more than one language ecosystem. Pick a maintained third-party OSS library from an ecosystem not represented by the live Sourcey docs proof already on the board, then publish the generated docs on the target project's own real site or repo, or a credible durable home the project would adopt (not a personal or throwaway host).

Deliverable:A live Sourcey-generated docs site for a maintained OSS library in a second ecosystem, with public_url, evidence_json, receipt_ref, and report artifacts.

Acceptance
  • The governed receipt or validation run uses runx CLI 0.6.6 or newer; evidence_json.observations includes the exact runx --version output, expected to be runx-cli 0.6.6 or newer.
  • The selected ecosystem differs from any live or accepted Sourcey docs bounty already visible on the board at claim time.
  • The target is maintained, third-party, permissively licensed, and pinned to a public commit.
  • The target has at least 15 public APIs, endpoints, modules, examples, or configuration concepts worth documenting.
  • The docs live on the target project's real site/repo, or a credible durable home the project owns or would adopt; a personal or throwaway host (a <handle>.github.io page, a demo/docs-hub org, a test account) does not qualify, however clean the pages are.
  • public_url loads for a stranger and shows generated navigation.
  • The docs include source-mapped generated pages, not only a landing page.
  • evidence_json observations include ecosystem, repo, commit, license, adapter, command/config, page list, and gaps.
